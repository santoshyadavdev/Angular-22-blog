WebMCP Tools Support

Angular 22 introduces experimental WebMCP support, enabling AI agents to invoke tools registered in your Angular app via the WebMCP protocol.

🧪
Experimental API — These APIs are prefixed with Experimental and may change in future versions.

1. ToolDescriptor Interface

Every WebMCP tool is defined by a ToolDescriptor with a name, description, a JSON Schema inputSchema, and an execute callback. The Client provides an AbortSignal for cancellation.

name Unique identifier the agent uses to call this tool
description Helps the agent decide when to use the tool
inputSchema JSON Schema — TypeScript infers arg types from it
execute Callback that runs in an injection context
client.signal AbortSignal for cancelling long operations 
{{ toolDescriptorCode }}

2. provideExperimentalWebMcpTools()

Register tools at the application level in app.config.ts. Tools are available for the entire lifetime of the app. The execute callback runs in the application's injection context, so you can use inject().

1 Define tools with provideExperimentalWebMcpTools()
2 An MCP-compatible agent connects to the app
3 Agent discovers and invokes tools via the WebMCP protocol
4 Angular runs the execute function and returns the result 
{{ provideToolsCode }}

3. declareExperimentalWebMcpTool()

Register a tool dynamically inside a component or service. The tool is tied to the injection context — it's automatically unregistered when the component is destroyed. This is ideal for context-specific tools that should only be available while a certain view is active.

Component Created → Tool registered & available to agents
Component Destroyed → Tool automatically unregistered 
{{ declareToolCode }}

4. Route-Level Tools

Since provideExperimentalWebMcpTools() returns EnvironmentProviders, you can scope tools to specific routes. The tools are only available when the route is active — perfect for admin panels, dashboards, or feature-specific AI capabilities. 

{{ routeLevelCode }}

Key Points

  • Experimental — APIs are prefixed with Experimental and may change.
  • Type-safe — TypeScript infers execute argument types from the inputSchema JSON Schema via InferArgsFromInputSchema.
  • Injection context — execute runs in the associated injector's context, so inject() works inside it.
  • Automatic cleanup — declareExperimentalWebMcpTool unregisters the tool when the injection context is destroyed.
  • Cancellation — The Client provides an AbortSignal to cancel long-running operations.
  • Scoped registration — Use at the app level, route level, or component level depending on when the tool should be available.
